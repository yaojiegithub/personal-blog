婚前背景调查员：一个从业者的反思


大家好，我叫小林制药，一个恋爱/婚前背景调查员。今天想给大家讲讲我的职业，和这个职业给我的一些感悟。

婚前背景调查算是一个比较有意思和少见的工作。我一开始原本是一个对研究开源信息调查感兴趣的程序员。开源信息调查是一门很有用的综合性学科，通俗叫“人肉搜索”。主要是用已经公开的网络信息和印刷信息等种种不限形式并且已经公开过的信息，利用各种信息平台和专业信息搜集工具，做有针对性地调查。调查对象/主题可以是人，也可以是公司/社会团体。常用的平台和工具包括微信、脸书等社交媒体，政府户籍户口记录、商业记录、被公开和没被公开的消费记录，等等明网暗网能找到的各种信息。

因为我常驻北美，对美国的华人生活的方方面面比较了解，所以我的客户多数是在北美工作和生活的华人，以女性居多，偶尔也有男性。我的委托人一般是一个已经和对方约会一段时间，非常慎重的准备正式步入一段关系或者婚姻的人，委托我调查她男朋友/女朋友。客户调查目的是希望在正式结婚之前或者正式开始一段关系之前，做一下背景调查，希望自己看中的是一个名实相符，值得托付的恋人。偶尔也会遇到非常爱慕前男友/女友的客户，分手之后仍然非常想知道自己爱过的这个人的方方面面，通过专业手段更多的了解对方，试试看能不能找到自己不知道的情况的。

我调查的方面包括一个人的教育和工作历史，年龄和出生日期，婚姻记录，犯罪记录，联系方式，兴趣爱好，圈子，收入，经济状况，名下资产（限少数实体资产），家庭成员情况。一般来说，被调查人和被调查家庭成员如果在美工作生活五年以上或公民，公开的信息会更多一些，不过具体也看个人情况。

婚前背景调查在欧美国家的其实已经很成熟了，在欧美的高净值人群结婚求职的时候其实很普遍，但是对绝大多数华人来说，还是个新鲜事物。这个行业的从业者里面很多人都身兼数职，全职做这个行业的基本上传统私家侦探最多。黑客、离婚律师、猎头、信用调查机构、讨债机构、房地产从业者等等这些人的工作内容本身很大程度就是信息搜集，他们工作很重要的一部分也是用一些手法和信息平台做个人全面或者片面的背景调查。我兼职做这个行业，使用的绝大多数信息搜集工具和手法，多数是自己看书上网找教程学和总结的。我使用的方法和平台资源，绝大多数来自上述提到的这些行业。

我接到过少数商业调查的委托，绝大对数是针对个人的背景调查的委托。因为我对人，和不同人的生活经历更感兴趣。因为每个人都有完全不同的生活，收集到的信息和搜集信息的方式也会非常不一样。调查的过程往往是个创造性解决问题、用已知信息推导未知信息的过程，我觉得最有挑战性也最有趣，而且是自己喜欢和擅长的方面，所以渐渐做恋爱/婚前个人背景调查这方面比较多。

因为这一行黑白兼半，不过我个人做背景调查的内容和有严格的个人范围。一、我做调查只用已经被公开的开源信息（Open Source Intelligence)，即我不入侵和破解被调查人还没有被公开/破解过的设备和密码。二、我不做私家侦探，不提供跟踪偷拍监听业务。三、我只做背景调查，不泄露极端敏感的个人隐私和商业机密 四、只做恋爱/婚前背景调查，不捉小三不接婚外情调查业务。

之所以给自己划定这样的范围，一是因为我做这一行最主要的原因是自己很喜欢学习信息挖掘的技巧和手法，想让自己从实战得到更多锻炼，想把精力和时间尽量多用在信息挖掘和整理这个环节上，不想花太多精力处理自己并不太感兴趣的方面。二是因为想用自己会的知识和技能，实实在在的帮到一些人。三是想要规避风险。

人在社会生活，凭本事大小，或许遵守社会的规律的程度也许可以有一点不一样。我只是一个很珍惜自己现有生活的普通人，不是私家侦探也不算黑客，自问还没有可以不守规则的本事，只希望能尽量在合理合法的范围内，做自己喜欢的事。所以我和其他从业者最大的不同之处，就在我只用已公开的开源信息（OSINT)，不黑目标设备也不强行破解密码。不过这只是我自己生活的方式，并不是这世上唯一处事准则。每个从业者有他们需要从事的工作和他们自己的的处事风格，我非常尊重这种不同。大家互相理解，在自己舒适的范围内做事就好。


那么问题来了：我到底怎么查找一个人的信息呢？

基本步骤：
一、从委托人处得到目标调查人的基本信息
二、提炼关键信息点（种子），找到区分要调查的目标A和其他同名同姓的人不同特点的关键信息
三、利用种子信息点，去多种平台收集犯罪，结婚，经济情况等等各个方面的信息，相互关联，相互印证，步步推开，每个方面详细推敲梳理
四、通过多种途径手段，信息之间互相印证，去伪存真，验证关键信息真实性
五、写报告，私密方式发给委托人

举个具体的例子：
我的客户委托我背景调查她准老公张鹏（人名虚构）。张鹏来美国上学+工作了14年，33岁，现在美国东北部某主要城市做程序员经理。
 
我查张鹏，一般会从一个人的简历和领英页面（LinkedIn)和简历入手开始调查。在北美生活的绝大多数华人，多数都有在本地上过学/找工作/工作的经历。因为简历和领英上面可以获得非常多的关键信息（手机号，邮箱，曾用住址，上过的学校，工作实习经历，个人网站博客，论文曾获奖项等等），这些关键信息可以在后面的调查中作为种子信息，用来挖掘更多信息，相互印证和缩小包围圈。
 
下一步，从各种搜索引擎上面搜索从第一步的种子信息。比如，用专门搜索手机号码的反向搜索引擎，挖掘出更多和这个手机号相关联的信息。把第一步找到的所有种子信息全搜索一遍，把引申出的新信息记录好，作为新一轮种子信息，进一步搜索。
 
再下一步，按照（婚姻历史，犯罪历史，经济状况等等）不同类别的信息，分门别类地梳理查找感兴趣的方面。稍后我会拿我查找张鹏的婚姻历史的过程来具体举例。 
这里先给大家补充一点背景信息：
在美国，一个人的结婚记录是公开信息（不同州政策不同）。当一对新的夫妇想要结婚，他们会到他们所在的county所在的市政厅去找法官签署并申请结婚许可(marriage license)。County一种比市小一级的行政地区域划分，类似北京市有海淀区大兴区，county是一个城市下面的其中一个行政区/县，读者可以联想一下类似于北京市海淀区民政局的一个机构。等十到十五个工作日之后，去领取结婚证（certification of marriage), 领到结婚证之后，去州务卿办公室（secretary of state）做公证。因此，一个人的结婚证和结婚离婚历史，在美国绝大多数州（每个州政策不一样），是有据可查的公开信息。不过因为每个州政策不一样，公开信息的情况和有无网络数据库的情况不一致，查询起来很繁琐。而且美国这边每个城市下面的county行政区划分数量非常多，如果要查询一个人的婚姻信息，必须先知道他具体在哪个county申报的，如果在错误的county查询（比如对方在海淀区民政局登记结婚的，你去大兴区民政局的数据库里搜索，是搜索不到结果的），哪怕对方已经结了婚，也是找不到结婚信息的。
除了行政机构公开的结婚许可，一个人如果曾经结了婚，还有很多其他的蛛丝马迹可循。比如说，中国人在美国的大城市结婚，一般比较喜欢在郊区买个房子，如果一个人的曾用邮寄地址中（曾用邮政地址历史很好找到）名下有郊区的房子，那么这个人结过婚的可能性是比较高的。找到对方名下房子的地址以后，可以顺藤摸瓜搜索出曾经用过居住过同一个房子使用过同一个邮政地址的人。如果两个人曾经在同一个房子共同居住并使用同一个邮政地址过很长一段时间，那么他们要么是室友，要么就是在同居。下一步可以进一步核实两个人的关系。核实关系有很多途径，比如说查询另一个人的结婚地址，教育历史，朋友圈交集，社交网络发布的图片和社交状态更新等等。
那么我怎么具体来查找张鹏的婚姻历史呢？
我从张鹏发布在领英（LinkedIn)上的简历和社交网络上公布的教育经历和工作历史中了解到，他在换工作来到美国东北部之前，曾在中部的旧金山一所大学上过两年研究生和在芝加哥郊区某科技公司工作过四年时间。
下一步，我会用thatsthem.com或者类似的专门搜索人的搜索引擎来搜索，一般会出一些结果，根据简历上列出来的年龄手机号曾用地址和城市可以缩小包围圈。不过人搜索引擎结果一般不全面而且很多时候结果不准确，只能作为辅助。

我又郊区房子的地址信息反向搜索，找到了这个地址曾住过的人员。其中有一个中国女生的名字就是上面这个第二贷款人。我单独查询了一下这个女生的社交媒体尤其是她和张鹏可能有交集的时间范围。找到了很多生活照和几张结婚照，其中就有张鹏。
我进一步核实了一下两人社交网络朋友圈，发现有很大的时间，地点交集和共同好友交集。我在张鹏的个人网站上看到了几篇他写的旅游游记，从游记中设法找到了他几次出去旅行本人拍摄的几张原图。虽然图像本身只有风景和饭店食物等等，少有人物，但我从原图隐含的图片信息中（Meta data)，找到了图片拍摄的时间地点，和女生社交网络上公布的几次旅游出行地点和时间吻合。

我用同样的方式查找了女生发布在社交网络的活动，在他俩同居的一两年活动地点和时间基本上高度重合。至此张鹏和女生结过婚同过居实锤了。
近几年绝大多数社交媒体，比如微信脸书等主料社交媒体，处于安全考虑都会把用户上传的照片的隐含图片信息去掉，但是社交媒体之外的网站仍然有很多网站是不会去掉这些照片隐含信息的，可以用来辅助调查。
我又去查找了一下两个人的离婚记录，发现两个人结婚后大概一年左右就登记了离婚，然后张鹏又过了一年换工作到了另外一个城市，和女生就没少有交集了。但调查到此两个人结过婚同过居又在一年内离婚的事情基本证实了。
从调查中我能感受到张鹏曾经有意清理过他的社交媒体关于他结婚和前妻的这一段经历相关的照片和状态更新，因为他自己的社交媒体明显很干净没有任何结过婚的迹象。我后来在通过图片搜索，在调查过程中找到了几个张鹏在几个不同的美国婚恋网站中的个人资料，他都没有提到过自己曾经结过婚的事情。我感觉张鹏至少在社交媒体的层面是想要隐瞒自己曾经短暂的结过婚并且很快离过婚的事吧。可能处于保护自己的目的。
不过这个调查也告诉我们，即便一个人自己能删掉很多社交媒体，你也无法控制自己身边的人。比如在这个例子里面，张鹏虽然社交媒体上完全看不出来曾经结过婚，但是她前妻不一定会删过去的社交媒体。而且除社交媒体之外，任然有很多公开记录和线索，能让我们发现两个人结过婚的事情。
我不知道我的委托人是否知道张鹏曾经结过婚的事情，也不知道鹏是否在故意隐瞒他的这段经历，还是已经在恋爱过程中坦诚地告诉了我的委托人他了曾经结过婚并且离过婚的事情。我的工作只是如实呈现我找到的所有信息，让客户自己做决断，我也从来不会和我的客户核实这样的事情，因为是客户的隐私。
但是我作为一个从业者，想告诫一下想要结婚的男孩子和女孩子：在美国不管和中国人还是和美国人结婚，可以到对方生活过的county去查询一下结婚历史，犯罪历史，经济犯罪历史，交通违章历史等公开信息。其实一个人结过婚的历史比起经济犯罪和刑事犯罪的历史比起来根本不够看的。我曾经碰到过身边的情侣朋友，一方（美国人）对另一方隐瞒过去的经历的，过了很久结了婚有了孩子，才发现对方曾经有过欠信用卡债的经济犯罪记录。欠钱要夫妻一起还钱还是小事，主要是和有这样不良经济习惯的男人结婚未来要生活在一起，自己和家人都比较闹心。如果婚前能自己动手查一下对方犯罪历史，查一下，不吃亏。

跟大家分享几点我做这一行的想法：
想法一：
我做信息搜索挖掘这一行，学到的很重要的一件事就是：有时候宁愿做决策的时候没有可用信息，也没有做决策的时候用错误和不准确的信息所带来的危害大。
我上面列举的所有网站，和近几年美国有很多免费和付费的个人背景调查搜索引擎，情一定谨慎对待从上面获得的情报。上面的情报不一定准确，甚至常常不准确。请大家一定不要用付费的个人背景调查网站比如Spokeo之类的。美国有很多背景调查的网站，搜索一个人名出来一些结果，如果是免费的可以看一下，跟你能够确定的信息做比对验证真实性，不建议大家花钱买结果，比如 1，2，3 网站，很多时候花了前也不会给你更多结果，他们网站nanner设定就是故意十五分钟才出一点点结果给你看，增加你的时间沉默成本，增加你的期待值， 实在等不了了，掏钱买，买到的信息比之前banner给你看的报告很少有任何真正有效的信息，基本9成以上都是白掏钱，不要问我怎么知道的：）这类网站常常更换，往往免费网站马上变成收费的了。
有些个人背景调查网站我觉得拿来背调美国人还是比较有用和准确的，但是搜索中国人的话准确率比较低。美国人开发的个人搜索引擎有个很大的弊端，就是他们分不清中国人同一个发音谁是谁。假如说我现在要找一个人，叫王一凡，英文拼音都是Yifan Wang. 但是同一个英文拼音可以对应王义樊，望一帆等等不同的中文字和发音。虽说有其他的年龄，城市，住址等等信息可以帮助缩小包围圈，但是一个英文的个人背调网站，只是单纯的把找的到的两百多个结果摆在你面前，给你出一个报告，这份报告包含一份他们觉得和可能性最高的Yifan Wang的个人资料，这一份报告里面的这一个人，甚至不一定是你要找的那一个人，但是会对没有专业背调知识的人会有非常大的误导性。类似这样，一个美国人开发的自动算法五分钟出结果的关于一个中国人的报告，往往信息很不准确。不过也有相对质量比较好一些的平台，比如说Thomson Reuters CLEAR他们的背景调查平台，就还不错，是销售给美国法律执行部门的，个人公司也可以注册。不过说到底，一个最有效信息挖掘，需要一个了解在北美中国人生活的开源信息社会工程师，去层层搜索，去伪存真。
想法二：
不要基于有限的数据，去想想和假设一些东西，基于自己的想象和假设去做决定往往很危险。如果了解一个人，甚至一个人的某一个具体方面，情尽量全面而客观的了解，而不是片面的去了解。片面的条件往往会导致片面的假设，基于片面假设去做决定，决定往往是错的。
我举个具体的例子：
如果你是个女孩子，一心想嫁给有钱人，请问下面ABC三个人里面，你想嫁给谁？
A。全职程序员，美国某排名100以外学校计算机系毕业。爱好事打游戏和写C语言小程序，美国人。名下没有房子，一辆二手经济轿车。父母是教师和工程师。
B. 中国人，研究生在读。清华毕业，父母都是清华教授，北京有房。本科清华毕业，来美国上过四年医学院，芝加哥大学艺术系第二个硕士在读，在芝加哥市中心有独栋别墅。开奔驰。爱好是音乐和话剧。B来美国16年了。
C。 中国人，美国某排名30的学校本科生在读。本科出国，自己在本科的学校承包了一个中餐厅自己做老板。开保时捷。其他名下资产不清楚。爱好看娱乐节目和Party.来美国5年了。
如果你选，只知道上述条件，你会选谁呢？

如果加上下面这些条件，你会选谁呢？
A。 A非常喜欢信息安全，平时花很多时间研究写代码和网络安全并且在全职之外帮一些公司找系统漏洞和做安全渗透测试。他去年在赏金系统漏洞猎人的专业黑客平台上，去年他自己靠帮公司挖漏洞挣了六位数（美元）。如果A想跳槽一年工资翻一倍很正常，但是他不想跳槽，因为喜欢自己现在的工作有很多时间不太累，让他有很多个人时间做想做的事情。
B。 B家里条件曾经条件非常好，但是B最大的毛病是被父母娇惯的花钱大手大脚而且并不会自己挣钱。他现在在芝大读研究生一年税后只有四万块钱收入不到。他来美国留学十多年花了家里很多积蓄。虽然在芝加哥有房产，但是他并不是很会理财，也没有其他收入来源。虽然父母在北京养老金都不少，但是他父母已经快七十岁了，有很多医疗支出，自顾不暇。他在芝加哥市中心的房子很大，是他十年前刚到美国的时候父母给他买的。但是每年他自己这点收入，为了维持他的文化艺术需求和房子的维护房产税等等，他在生活上必须省吃俭用。芝加哥的冬天很冷，但是他房子很大，开空调一个月能花两百美元电费。为了省电费，他冬天在家都是穿羽绒服的，不敢开暖气。
C。 C同学的饭店经营的头两年年非常不错，但是第三年饭店状态不太好，另一个和他合伙做生意的同学不跟他合伙了。出事之后大家才知道，原来C同学在学校只上了两个学期就不上了，他用家里给他的学费买了保时捷和跟同学合伙开了之前的饭店而且还骗走了同学的钱跑路了。之前饭店经营好的时候，他在学校很出名，也有很多女生曾经很喜欢他。自从他卷钱跑路并且被警察以诈骗罪通缉之后，大家再也没有在校园里见过他。

上面的三个人，都是我自己曾经认识过的朋友，我只更改了一些细节。举这个例子，只是想告诉大家：人是复杂而多面的生物，并且一个人的情况是经常在变动的。如果你根据之前的条件，在跟三位男生交往过程中，产生了出了B或者C比A更有钱的印象或者假设，并且依据这个印象/假设来决定嫁给谁。这个假设本来就是错的，如果你依据错误的假设，嫁给了B或者C，那么这个决定也是错的。我真的不支持女孩子只根据男生有没有钱决定是否嫁给对方。我自己这么多年在留学圈子里面其实看过很多人的起起伏伏。一个人的价值，长期来看是会均值回归的。即使是考虑对方的经济条件，多了解一个人的思想格局，做事方式，他所从事的工作和行业，理财观念，远比他现在开什么车，名下有几套房这些来的有用的多。

不要走马观花/sterotype ,用一两条信息（比如仅仅一个人的资产状况）来决定开始或者结束一段关系或者去衡量一个人，”assume”一些东西，assumption不一定成立。人是复杂的生物，如果是真诚的恋爱和结婚的话，请静下心，多花些时间精力，从生活的点点滴滴，去了解一个人。有车和房的，不一定是你的知心人，暂时物质条件或者某些方面有欠缺的，只要不是关键deal breaker 请给对方也给自己一个机会，好好把对方当成一个人，一个朋友一样，去了解去磨合。付出真心和时间虽然不一定都能换到真心，人生在世，遇到各种奇葩骗子外加不适的人的几率，往往远高于遇到适合自己的知心人的几率。但对方是一个怎么样的人，和他/她朝夕相处的你，其实最了解。信息是非常有效的决策工具，也是很可怕的恋爱终结武器，请谨慎的使用情报。
